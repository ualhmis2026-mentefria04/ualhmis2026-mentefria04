## 🚀 Nuestro Despliegue en GitHub Pages

En este proyecto, nuestro equipo está desarrollando de forma colaborativa un blog estático utilizando **Jekyll**. El primer paso crucial ha sido lograr que nuestro código local sea visible para todo el mundo.

Para ello, hemos utilizado **GitHub Pages**, una herramienta nativa de GitHub que nos permite servir sitios web estáticos directamente desde nuestro repositorio.

### ¿Qué hemos conseguido?

Hasta ahora hemos asentado unas bases muy sólidas de ingeniería de software:

1. **Estructura base:** Creamos el archivo `_config.yml` y la carpeta `_posts` para las entradas.
2. **GitHub Flow:** Protegimos la rama `main` para evitar commits directos. Todo el trabajo se realiza mediante ramas secundarias y *Pull Requests*, asegurando que el código se revisa antes de integrarse.
3. **Puesta en producción:** Configuramos la sección *Pages* en los ajustes de nuestro repositorio para que la web se buildee sola.
