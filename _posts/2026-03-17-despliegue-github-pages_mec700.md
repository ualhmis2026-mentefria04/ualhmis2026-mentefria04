# 🚀 Nuestro Despliegue en GitHub Pages

[cite_start]En este proyecto, nuestro equipo está desarrollando de forma colaborativa un blog estático utilizando **Jekyll**[cite: 51]. El primer paso crucial ha sido lograr que nuestro código local sea visible para todo el mundo. 

[cite_start]Para ello, hemos utilizado **GitHub Pages**, una herramienta nativa de GitHub que nos permite servir sitios web estáticos directamente desde nuestro repositorio[cite: 85].

## ¿Qué hemos conseguido?

Hasta ahora hemos asentado unas bases muy sólidas de ingeniería de software:
1. **Estructura base:** Creamos el archivo `_config.yml` y la carpeta `_posts` para las entradas.
2. [cite_start]**GitHub Flow:** Protegimos la rama `main` para evitar commits directos[cite: 41]. [cite_start]Todo el trabajo se realiza mediante ramas secundarias y *Pull Requests*, asegurando que el código se revisa antes de integrarse[cite: 40, 44].
3. [cite_start]**Puesta en producción:** Configuramos la sección *Pages* en los ajustes de nuestro repositorio para que la web se buildee sola[cite: 90, 92, 93, 94].
